Chapter 01

为什么选 OpenClaw

先搞清一个核心概念：对话 vs 执行

Claude、ChatGPT、Gemini 本质上都是对话工具——你问它一个问题，它给你一段文字回复。不管回复多精彩，它都没有"手"，不能替你操作任何东西。

OpenClaw 是执行工具——它不只是回答你的问题，它能真的去做事：发邮件、管文件、跑代码、搜信息、操作浏览器、在你睡觉的时候自动完成任务。

打个比方：ChatGPT 是你的顾问（帮你想），OpenClaw 是你的助理（帮你干）。一个给建议，一个出结果。

Chapter 02

怎么用效果最好 — 让 AI Agent 真正帮你干活

核心原则：把 OpenClaw 当员工，不是当工具

大多数人用 AI 的方式是"一问一答"——问完就走。这就像雇了个人，但只让他回答问题，从不分配任务。OpenClaw 的正确用法是委派任务，像管理一个下属一样管理它。

错误用法："帮我查一下最近的中老年教育市场报告"

正确用法："每天早上 9 点自动搜索中老年教育行业新闻，整理成简报，推送到我的飞书"

五个效率最大化的关键方法

给它"人设"——写好 System Prompt

在 OpenClaw 的配置文件中写清楚它的身份和职责。比如："你是开开华彩的运营助理，主要职责包括：合伙人数据监控、竞品追踪、内部结算核对。"写得越具体，它干活越靠谱。

用 Skills 给它"技能"

OpenClaw 有一个 ClawHub 技能市场，包含 100+ 预配置技能。你可以让它搜索并安装你需要的技能，比如 Web 搜索、文件管理、日历操作等。

开启 Memory——让它记住上下文

配置中开启 Memory 功能后，OpenClaw 会记住之前的对话和任务上下文。它能持续积累对你业务的理解，越用越好用。

设定定时任务——让它自动运行

OpenClaw 支持 cron 定时任务和 heartbeat 心跳检测。你可以设定每天/每周/每月自动执行的任务。

多渠道交互

OpenClaw 支持飞书、钉钉、微信、Telegram、Discord 等 20+ 消息平台。把它接入你日常用的办公工具，用自然语言随时给它布置任务。

部署步骤

安装 Node.js 22

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - sudo apt-get install -y nodejs node -v

安装 OpenClaw

sudo npm install -g openclaw@latest openclaw --version

运行引导配置

openclaw onboard

启动并后台运行

screen -S openclaw openclaw gateway start # 按 Ctrl+A 然后按 D 退出 screen
验证成功：打开飞书，给机器人发一条消息"你好"。如果它回复了，恭喜，你的 AI Agent 已经上线了！
Chapter 06

常见问题与避坑指南

安全第一

🔒 权限最小化

不要用 root 用户运行 OpenClaw。创建一个专用用户，只给它需要的最低权限。

🛡️ 审慎安装 Skills

ClawHub 上的第三方 Skills 可能包含恶意代码。只安装来源可信的 Skill。

🔑 API Key 安全

不要把 API Key 写在代码里或分享给别人。用 OpenClaw 的 auth profiles 功能存储。

🌐 网络隔离

如果部署在云服务器上，配置防火墙，只开放必要端口。
